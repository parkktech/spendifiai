  <action>
    **StoreAnnotationRequest** (`app/Http/Requests/StoreAnnotationRequest.php`):
    - Rules: body (required, string, max:2000), parent_id (nullable, integer, exists:document_annotations,id)
    - authorize(): true (policy check happens in controller)

    **StoreDocumentRequestRequest** (`app/Http/Requests/StoreDocumentRequestRequest.php`):
    - Rules: description (required, string, max:1000), tax_year (nullable, integer, min:2000, max:2099), category (nullable, string, in: TaxDocumentCategory enum values)
    - authorize(): user is accountant type

    **TaxDocumentPolicy** — Extend with two new methods:
    - `annotate(User $user, TaxDocument $document)`: Same logic as `view()` -- owner or linked accountant
    - `requestDocument(User $user)`: User must be accountant type. (No document param needed -- it's about creating a request.)

    **DocumentAnnotationController** (`app/Http/Controllers/Api/DocumentAnnotationController.php`):
    - `index(Request, TaxDocument $document)`: Authorize view on document. Return top-level annotations (parent_id null) with nested replies (eager load author, replies.author). Max depth 2 levels. Return JSON array.
    - `store(StoreAnnotationRequest, TaxDocument $document)`: Authorize annotate on document. Create annotation with user_id from auth. If parent_id provided, verify parent belongs to same document. Queue AnnotationNotifyMail to the OTHER party (if author is accountant, notify document owner; if author is client, notify all linked accountants). Log audit entry 'annotation_created' via TaxVaultAuditService. Return created annotation with author (201).

    **DocumentRequestController** (`app/Http/Controllers/Api/DocumentRequestController.php`):
    - `index(Request, User $client)`: Verify accountant is linked to client (via AccountantClient::where). Return all requests for client from accountant's firm, ordered by created_at desc.
    - `store(StoreDocumentRequestRequest, User $client)`: Verify accountant is linked to client. Create DocumentRequest with accounting_firm_id from user's firm, accountant_id from auth, client_id from route. Queue DocumentRequestMail to client. Return created request (201).
    - `dismiss(Request, DocumentRequest $request)`: Verify request belongs to accountant's firm. Update status to 'dismissed'. Return 200.
    - `myRequests(Request)`: For client-facing route. Return all pending DocumentRequests for auth user, with accountant name eager loaded. Used by client vault to show alerts.

    Use `Mail::to()->queue()` for all email sends (not `send()`).

    **Routes** — Add to `routes/api.php`:

    ```php
    Route::get('/documents/{document}/annotations', [DocumentAnnotationController::class, 'index']);
    Route::post('/documents/{document}/annotations', [DocumentAnnotationController::class, 'store']);

    // Document requests
    Route::get('/clients/{client}/requests', [DocumentRequestController::class, 'index']);
    Route::post('/clients/{client}/requests', [DocumentRequestController::class, 'store']);
    Route::patch('/requests/{request}/dismiss', [DocumentRequestController::class, 'dismiss']);
    ```

    In the authenticated user group (`v1` with 'throttle:120,1'):
    ```php
    // Client can view their document requests
    Route::get('/document-requests', [DocumentRequestController::class, 'myRequests']);

    // Client annotations on own documents
    Route::get('/tax-vault/documents/{document}/annotations', [DocumentAnnotationController::class, 'index']);
    Route::post('/tax-vault/documents/{document}/annotations', [DocumentAnnotationController::class, 'store']);
    ```

    **Auto-fulfillment** — In TaxDocumentController::store() (or the appropriate upload handler), AFTER successful document upload and classification:
    - Query DocumentRequest::where('client_id', $document->user_id)->where('status', 'pending')
    - Filter: if request has tax_year, must match document tax_year. If request has category, must match document category value.
    - For each matching request: update status to 'uploaded', set fulfilled_document_id to document id
    - Queue RequestFulfilledMail to request's accountant

    Also in TaxDocumentController::store(), queue DocumentUploadedMail to all linked accountants (via AccountantClient::where('client_id', $user->id)->active()->get() -> each accountant).
