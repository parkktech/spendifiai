    **TaxDocumentUploadRequest** (`app/Http/Requests/TaxDocumentUploadRequest.php`):
    - `authorize()`: return true (policy handles document-level auth)
    - `rules()`: `'file' => 'required|file|mimes:pdf,jpg,jpeg,png|max:102400'` (100MB), `'tax_year' => 'required|integer|min:2000|max:' . (date('Y') + 1)` -- allow current year + 1 for future filings

    **StorageConfigRequest** (`app/Http/Requests/Admin/StorageConfigRequest.php`):
    - `authorize()`: `return $this->user()->user_type === 'admin';`
    - `rules()`: `'driver' => 'required|in:local,s3'`, `'s3_bucket' => 'required_if:driver,s3|string'`, `'s3_region' => 'required_if:driver,s3|string'`, `'s3_key' => 'required_if:driver,s3|string'`, `'s3_secret' => 'required_if:driver,s3|string'`

    **TaxDocumentResource** (`app/Http/Resources/TaxDocumentResource.php`):
    Returns: id, original_filename, mime_type, file_size, file_hash, tax_year, category (use category enum label if set), status, classification_confidence, created_at, updated_at, signed_url (call TaxVaultStorageService::getSignedUrl). Do NOT include stored_path or disk.

    **TaxVaultAuditLogResource** (`app/Http/Resources/TaxVaultAuditLogResource.php`):
    Returns: id, action, user (id + name), metadata, created_at. Conditionally include ip_address and user_agent only when `$this->resource->ip_address !== null` (Super Admin makes them visible before serialization).

    **TaxDocumentPolicy** (`app/Policies/TaxDocumentPolicy.php`):
    - `viewAny(User $user)`: always true (filtered by user scope)
    - `view(User $user, TaxDocument $document)`: true if `$user->id === $document->user_id` OR user is accountant linked to document owner via AccountantClient relationship
    - `create(User $user)`: always true for authenticated users
    - `delete(User $user, TaxDocument $document)`: true only if `$user->id === $document->user_id` (accountants cannot delete client docs)
    - `purge(User $user)`: true only if `$user->user_type === 'admin'` -- admin-only hard delete
    - `viewAuditLog(User $user, TaxDocument $document)`: same as view() -- owner and linked accountant

    **MigrateStorageJob** (`app/Jobs/MigrateStorageJob.php`):
    - Implements ShouldQueue
    - Constructor: `string $targetDisk`
    - `handle(TaxVaultStorageService $storageService)`: Queries all TaxDocuments where `disk != $targetDisk`. For each, calls `$storageService->migrateDocument($document, $this->targetDisk)`. Uses chunking (50 at a time) to avoid memory issues. Stores progress in cache key `storage-migration-progress` as `['total' => int, 'migrated' => int, 'status' => 'running'|'complete'|'failed']`.
    - `failed(\Throwable $e)`: Sets cache status to 'failed' with error message.

  <action>
    **TaxDocumentController** (`app/Http/Controllers/Api/TaxDocumentController.php`):
    Inject TaxVaultStorageService and TaxVaultAuditService via constructor.

    - `index(Request $request)`: List user's documents. Accept `?year=2025` query param. Use `$request->user()->taxDocuments()->byYear($year)->latest()->get()`. Return TaxDocumentResource::collection. The user relationship on User model needs to be added: `taxDocuments() hasMany TaxDocument`. Log 'list' action in audit (do NOT log list actions -- too noisy. Only log view/download/upload/delete).
    - `store(TaxDocumentUploadRequest $request)`: Upload document. Call `$storageService->store($request->file('file'), ...)`. Create TaxDocument record with status 'upload'. Log 'upload' action via audit service. Return TaxDocumentResource with 201 status.
    - `show(TaxDocument $document)`: Authorize via policy `view`. Log 'view' action. Return TaxDocumentResource (includes signed_url).
    - `download(TaxDocument $document)`: Authorize via policy `view`. Log 'download' action. Return redirect to signed URL or stream the file content.
    - `destroy(TaxDocument $document)`: Authorize via policy `delete`. Log 'delete' action. Soft-delete the document (do NOT delete the file -- admin purge handles that). Return 204.
    - `purge(TaxDocument $document)`: **Admin-only hard delete.** Authorize via policy `purge`. Use `TaxDocument::withTrashed()->findOrFail($document)` to find the document (may already be soft-deleted). Call `$storageService->delete($document)` to remove the file from storage. Log 'purge' action via audit service. Then call `$document->forceDelete()` to permanently remove the database record. Return 204.

    **Important**: Add `taxDocuments()` relationship to User model: `public function taxDocuments(): HasMany { return $this->hasMany(TaxDocument::class); }`

    **TaxVaultAuditController** (`app/Http/Controllers/Api/TaxVaultAuditController.php`):
    Inject TaxVaultAuditService.

    - `index(TaxDocument $document, Request $request)`: Authorize via TaxDocumentPolicy `viewAuditLog`. Call `$auditService->getLogForDocument($document, $request->user())`. Return TaxVaultAuditLogResource::collection.
    - `verifyChain(TaxDocument $document, Request $request)`: Only for admin users (`abort_unless($request->user()->user_type === 'admin', 403)`). Call `$auditService->verifyChain($document)`. Return JSON result.

    **StorageConfigController** (`app/Http/Controllers/Api/StorageConfigController.php`):
    Inject TaxVaultStorageService.

    - `show(Request $request)`: Admin only. Return current storage config (driver, stats from `$storageService->getStorageStats()`, migration progress from cache).
    - `update(StorageConfigRequest $request)`: Update config values. For S3 credentials, encrypt and store in database settings table or use a simple `settings` JSON file. Actually, use the `cache` or a `Setting` model -- BUT since this project doesn't have a settings model, store S3 config in `.env` is not runtime-safe. Instead: store encrypted S3 credentials in a `vault_storage_settings` cache key with no expiry, and update `config('spendifiai.vault.storage_driver')` at runtime. Document this approach in code comments.
    - `testConnection(Request $request)`: Admin only. Call `$storageService->testS3Connection()`. Return success/failure JSON.
    - `migrate(Request $request)`: Admin only. Validate target disk. Dispatch MigrateStorageJob. Return 202 accepted with migration ID.
    - `migrationStatus(Request $request)`: Return migration progress from cache key.
