    ```typescript
    export interface StorageConfig {
      driver: 'local' | 's3';
      stats: {
        total_documents: number;
        total_size_bytes: number;
        active_driver: string;
      };
      migration_progress: {
        total: number;
        migrated: number;
        status: 'idle' | 'running' | 'complete' | 'failed';
        error?: string;
      } | null;
    }
    ```

    **Admin/Storage.tsx** page:
    Layout: AppLayout with title "Storage Settings". Follow existing Admin/Dashboard.tsx patterns for data fetching (axios + useState).

    Sections:
    1. **Summary Stats Row** (3 StatCards):
       - Total Documents (count)
       - Total Storage Used (formatted bytes -> MB/GB)
       - Active Driver (local/s3 with colored indicator)

    2. **Storage Driver Toggle**:
       - Two radio/toggle buttons: "Local Filesystem" and "Amazon S3"
       - Current driver highlighted with sw-accent
       - Toggle is DISABLED when migration is running (check migration_progress.status === 'running')
       - Changing driver triggers save via PUT /api/v1/admin/storage

    3. **S3 Configuration Form** (visible only when S3 is selected or being configured):
       - Fields: Bucket Name, Region (dropdown with common AWS regions), Access Key ID, Secret Access Key (password input)
       - "Test Connection" button -- calls POST /api/v1/admin/storage/test
       - Test result shown inline: green checkmark "Connection successful" or red X with error message
       - "Save Configuration" button -- only enabled after successful test (per user decision: "credentials only saveable after successful test")
       - Fields styled with `sw-card` background, `sw-border` borders

    4. **Document Migration Section**:
       - "Migrate Documents" button with confirmation dialog (ConfirmDialog component)
       - Button text: "Migrate {N} documents to {targetDriver}"
       - Progress bar during migration: `bg-sw-accent` fill with percentage text "Migrating 45/120 documents..."
       - Poll migration status every 2 seconds during active migration via GET /api/v1/admin/storage/migration-status using setInterval
       - On completion: show success message, refresh stats
       - On failure: show error message with red styling

    Fetch initial data on mount via GET /api/v1/admin/storage. Use axios (following Admin/Dashboard.tsx pattern, not useApi hook).

    **Design**: Match existing admin pages. Use `sw-card` for sections, `sw-border` for dividers, `sw-accent` for active states, `sw-text` for labels. Use lucide-react icons: HardDrive for local, Cloud for S3, RefreshCw for migration.
