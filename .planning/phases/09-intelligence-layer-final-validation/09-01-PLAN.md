  <name>Task 2: Create TaxDocumentIntelligenceService and API endpoint</name>
  <files>app/Services/AI/TaxDocumentIntelligenceService.php, app/Http/Controllers/Api/TaxDocumentController.php, routes/api.php</files>

1. Create app/Services/AI/TaxDocumentIntelligenceService.php following the TaxDocumentExtractorService pattern:

   **Public method:** `analyze(int $userId, int $taxYear): array`
   - Cache result with key `tax_intelligence_{$userId}_{$taxYear}` for config('spendifiai.intelligence.cache_hours') hours using Cache::remember()
   - Returns: `['missing_documents' => [...], 'anomalies' => [...], 'transaction_links' => [...]]`

   **Protected method:** `detectMissingDocuments(int $userId, int $taxYear): array`
   - Query income transactions (amount < 0) for user+year
   - Classify each transaction's income type using the same logic as IncomeDetectorService::classifyType() -- replicate the plaidTypeMap/plaidPrimaryMap/aiTypeMap/merchant heuristic cascade (don't inject IncomeDetectorService, as it has different public API; replicate the classification maps)
   - Add 'dividend' type: check plaid_detailed_category for 'INCOME_DIVIDENDS' specifically (IncomeDetectorService maps this to 'interest', but intelligence needs to distinguish)
   - Add mortgage detection for expenses: check plaid_category containing 'LOAN_PAYMENTS' or merchant containing 'MORTGAGE', 'HOME LOAN' for transactions with amount > 0
   - Group income by type and normalized merchant name (strip PAYROLL, DIRECT DEP, SALARY suffixes)
   - For each income type+merchant group, check if a matching TaxDocument exists (by category from config income_type_to_document map)
   - Only flag missing documents when total income from that source >= config min_income_threshold ($600)
   - Return array of: `['message' => 'Missing W-2 from {employer}', 'details' => 'Based on $X,XXX in employment income from {employer}, we expect a W-2.', 'category' => 'w2', 'severity' => 'warning', 'merchant' => '{employer}', 'total_amount' => float]`

   **Protected method:** `detectAnomalies(int $userId, int $taxYear): array`
   - Load TaxDocuments with status=Ready and non-null extracted_data
   - For W-2 docs: compare extracted wages field against deposit total from matching employer transactions. Note: net pay != gross wages (taxes withheld), so deposits will normally be LOWER than W-2 wages. Flag when deposits are significantly HIGHER than document amount (possible unreported income) or when deposits are less than 50% of document amount (possible missing account). Use anomaly_tolerance from config.
   - For 1099-NEC: compare nonemployee_compensation against contractor income deposits
   - For 1098: compare mortgage_interest against mortgage payment totals
   - Return array of: `['message' => '{type} mismatch for {entity}', 'details' => 'Human-readable explanation', 'severity' => 'info'|'warning'|'alert', 'document_id' => int]`
   - Severity: >20% variance = 'alert', >tolerance = 'warning', else skip

   **Protected method:** `linkTransactions(int $userId, int $taxYear): array`
   - For each TaxDocument with status=Ready, find matching transactions by:
     - W-2: employment income transactions where normalized merchant matches employer_name from extracted_data
     - 1099-NEC: contractor income matching merchant
     - 1099-INT: interest income transactions
     - 1098: mortgage payment transactions
   - Use sync() on the belongsToMany to persist links with link_reason
   - Return array of: `['document_id' => int, 'document_name' => string, 'transaction_count' => int, 'total_amount' => float]`

   **Cache invalidation:** Add static method `invalidateCache(int $userId, int $taxYear): void` that calls Cache::forget("tax_intelligence_{$userId}_{$taxYear}")

2. Add intelligence endpoint to TaxDocumentController:
   - Inject TaxDocumentIntelligenceService via constructor (add alongside existing services)
   - Add method: `public function intelligence(Request $request): JsonResponse`
   - Validate year param (integer, 2000-2099 range, default to current year)
   - Call service->analyze($request->user()->id, $year)
   - Return JSON response

3. Add cache invalidation to TaxDocumentController::store() method -- after successful upload, call TaxDocumentIntelligenceService::invalidateCache($request->user()->id, $document->tax_year)
